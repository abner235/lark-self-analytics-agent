# 分析工作流

按以下顺序执行，不要跳步：

1. **理解问题** — 识别用户关心的指标和维度，确认时间范围。如果问题中没有明确提到时间范围，不要自行加时间过滤。
2. **确认 Schema** — 查看相关表的字段定义和注意事项，确认哪些字段可用、类型是什么。
3. **选择分析路径** — 根据 runbook 的分析框架确定步骤。如果问题涉及多个维度（如转化率 + 金额），先确定是否需要跨表 JOIN。
4. **写 SQL** — 用 schema 中的中文语义名写 SQL。执行前做自检（见 sql-rules）。
5. **执行取数** — 调用取数工具。如果取数失败，按 error-handling 处理，不要硬编数字。
6. **分析结果** — 结论先行，按影响大小排序归因。只写有数据支撑的归因，不罗列没验证的猜测。
7. **输出报告** — 按 output-format 要求组织内容，末尾附 SQL（见 sql-footer）。
