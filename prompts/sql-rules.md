# SQL 规则与常见错误

每次写完 SQL、执行前，按以下规则自检。如果自检发现问题，先修正再执行，不要带着已知问题执行。

## SQL 层面

1. **字段名** — 只用 schema 中定义的语义名（中文），不要编造字段。
   ⚠️ 常见错误：猜测不存在的字段名。这是最常见的错误。

2. **聚合函数** — 必须匹配 schema 中 `type` 声明。`type: sum` → `SUM()`，`type: count` → `COUNT()`，`type: countDistinct` → `COUNT(DISTINCT ...)`。
   ⚠️ 常见错误：搞混聚合函数类型，导致数值偏差。

3. **漏斗阶段值** — 只能用 schema 中列出的英文值（created / kyc_passed / pay_submitted / pay_authorized / completed），不要用中文值。
   ⚠️ 常见错误：用中文值过滤漏斗阶段。

4. **跨表 JOIN** — 漏斗表每个 key 组合有 5 行（5 个阶段），直接 JOIN 会导致数据膨胀 5 倍。必须先用 CTE 把漏斗表聚合到目标维度，再 JOIN 订单表。
   ⚠️ 常见错误：直接 JOIN 漏斗表和订单表，不做预聚合。

5. **日期格式** — 用 `'YYYY-MM-DD'` 格式，不要用 `CURRENT_DATE` 或相对日期函数。
   ⚠️ 常见错误：用 `CURRENT_DATE`、`DATE_SUB()` 等函数。

6. **时间范围** — 用 `>`、`<`、`>=`、`<=` 过滤，不要用日期函数计算。问题中没有明确提到时间范围时，不要自行加时间过滤。

7. **嵌套聚合** — 需要嵌套聚合时用 `WITH` 语句（CTE），不要用子查询嵌套。

## 分析层面

8. **没有数据支撑的归因** — 归因必须有取到的数字佐证。「可能是因为 X」这种没跑数验证的猜测不要写。

9. **裸数字** — 数字必须带口径和时间范围。「转化率 50%」不行，「Banxa 6/28 整体转化率 50%（口径：completed/created）」才行。

10. **口径不确定但继续输出** — 口径不确定时，停下来说明，不要给一个看起来对的错数。
